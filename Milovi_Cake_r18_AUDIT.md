## 🎯 Ключевые исправления

### 1. Единая система ключей корзины (🔴 CRITICAL BUG FIX)
**Проблема:** `buildCartKey()` имела разную логику для hasMaxi/!hasMaxi — товары без hasMaxi получали ключи без суффикса `:regular`, а bento-режимы с hasMaxi получали `id:regular` или `id:maxi`. addToCart() создавал ключи вручную через конкатенацию, а не через buildCartKey(). Это приводило к рассинхронизации: товар, добавленный через каталог, имел ключ `"3"`, а тот же товар через калькулятор — `"3::"` или `"3:regular::"`.

**Решение (r18):** Единый формат `id:mode:fill:decor` для ВСЕХ товаров. 
- Введена `parseCartKey()` для обратной совместимости
- `addToCart()` теперь использует `buildCartKey()`
- `buildCartKey()` всегда включает mode, даже для обычных товаров
- Все операции с корзиной проходят через единый парсер

### 2. Theme Toggle SVG — конфликт классов (🔴 CRITICAL BUG FIX)
**Проблема:** В HTML иконки имеют классы `theme-icon--moon`/`theme-icon--sun`, а premium-overrides R3/R6 ссылаются на `.sun-icon`/`.moon-icon` которых нет. Также CSS везде мешает `display:none` с `position:absolute` от style.css, иконки то появляются, то исчезают.

**Решение (r18):** Упрощена CSS-логика — единая точка: `.theme-icon--moon` для луны, `.theme-icon--sun` для солнца. Убраны все дублирующие селекторы. CSS управляет только `display:block/none` в зависимости от `data-theme`.

### 3. Hero-мессенджеры: кольца vs пилюли (🟡 DESYNC FIX)
**Проблема:** premium-overrides R5 превращает hero-мессенджеры в пилюли (прямоугольные кнопки с цветом), а R9 снова превращает их в кольца (круглые SVG-иконки). Два патча перезаписывают друг друга в зависимости от порядка загрузки.

**Решение (r18):** Финализированы кольца (ring SVG) как единый стандарт для главной страницы. Убраны дублирующие R5-стили для hero-мессенджеров. Анимация ring→flat текста перенесена в JS и работает одинаково на всех страницах. Подъём flat-текста увеличен до -13px для читаемости.

### 4. Мёртвый код fill-open-ios (🟡 CLEANUP)
**Проблема:** closeFillPopup() всё ещё проверяет `document.body.classList.contains('fill-open-ios')` и чистит position:fixed, хотя эта механика заменена на lockBody() ещё в r12.

**Решение (r18):** Удалён dead code. Весь fill-popup использует lockBody()/unlockBody().

### 5. Переменная `calc-order-btn` в style.css vs premium-overrides (🟡 CONSISTENCY)
**Проблема:** style.css определяет `.calc-order-btn` с базовыми стилями (gold bg, border-radius), а premium-overrides R6 переопределяет его с тёмным градиентом. При этом в DOM нет элемента с этим классом (HTML использует `.calc-add-btn`).

**Решение (r18):** Оставлена только актуальная стилизация. Убраны дублирующие правила.

### 6. Синхронизация навигации (🟡 NAV FIX)
**Проблема:** scroll-spy в nav.js использует getElementById для секций, но на страницах пригородов (prigorody) секции имеют другие ID. IntersectionObserver не срабатывает, активное состояние не обновляется.

**Решение (r18):** В initCatalogNavScroll() добавлена проверка существования элементов перед observer. В nav.js scroll-spy теперь ищет `[data-section]` атрибуты, а не только ID.

## 🧪 Как тестировать

1. **Корзина**: Добавить бисквитный торт (1) → Добавить бенто (2) → Переключить на Макси → Добавить → Проверить, что в корзине 3 разных товара
2. **Калькулятор**: Переключить 3D торт → Выбрать начинку → Переключить обратно на бисквит → Убедиться, что начинки переключились правильно
3. **Theme toggle**: Нажать иконку → Сменить тему → Обновить страницу (F5) → Убедиться, что тема сохранилась
4. **Hero messengers**: Навести на WA/TG/MAX → Убедиться, что кольцевой текст поднимается, появляется плоская подпись
5. **Мобильная навигация**: Открыть бургер → Проверить все пункты → Закрыть → Открыть bottom nav "Ещё"
6. **Отзывы**: Прокрутить до секции → Дождаться анимации набора текста → Кликнуть на летающий скриншот

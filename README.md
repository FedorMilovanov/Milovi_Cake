# Milovi Cake — сайт кондитера

Сайт авторских тортов и десертов ручной работы [milovicake.ru](https://milovicake.ru).

## Структура проекта

```
/
├── index.html          # Главная страница
├── manifest.json       # PWA-манифест
├── sitemap.xml
├── favicon.svg
├── css/
│   └── style.css       # Основные стили (style.min.css генерируется при деплое)
├── js/
│   └── main.js         # Основной JS, завёрнут в IIFE (main.min.js — при деплое)
├── img/                # Изображения (WebP + og-cover.jpg для OG-превью)
└── prigorody/          # Лендинги по пригородам СПб
    ├── _template.html  # Единый шаблон с {{плейсхолдерами}}
    ├── _cities.csv     # Данные всех городов (1 строка = 1 пригород)
    ├── build.py        # Генератор: собирает index.html из шаблона + CSV
    ├── pushkin/
    ├── peterhof/
    ├── gatchina/
    └── ...             # index.html каждого пригорода — сгенерированный файл
```


## Пригороды — генератор страниц

Все 14 страниц пригородов генерируются из одного шаблона. Редактировать руками их не нужно.

**Изменить текст на всех страницах сразу** (например, обновить футер):
1. Открой `prigorody/_template.html`
2. Внеси правки
3. Запусти генератор: `python3 prigorody/build.py`

**Изменить данные конкретного города** (заголовок, SEO-текст и т.д.):
1. Открой `prigorody/_cities.csv`
2. Найди нужную строку и отредактируй поля
3. Запусти генератор: `python3 prigorody/build.py`

**Добавить новый пригород:**
1. Добавь строку в `prigorody/_cities.csv` по аналогии с остальными
2. Запусти генератор — папка и файл создадутся автоматически
3. Добавь новый город в `sitemap.xml`

## Разработка

Статический сайт без сборщика. Открывай `index.html` напрямую в браузере или через локальный сервер:

```bash
npx serve .
# или
python3 -m http.server 8080
```

## Деплой

Сайт хостится на GitHub Pages. Пуш в `main` → автодеплой.

Перед пушем минифицировать CSS и JS:

```bash
npx cleancss -o css/style.min.css css/style.css
npx terser js/main.js -o js/main.min.js --compress --mangle
```

> **Примечание:** `.min`-файлы не хранятся в репозитории (см. `.gitignore`).
